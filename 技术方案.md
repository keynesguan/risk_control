### 1.2 技术栈



| 层次           | 技术选型                                                | 说明                                   |
| ------------ | --------------------------------------------------- | ------------------------------------ |
| **后端框架**     | Python 3.10+ / FastAPI                              | 异步 API、自动文档、类型校验（Pydantic）           |
| **关系型数据库**   | PostgreSQL 14+                                      | 支持 JSONB（规则存储）、窗口函数（成绩单聚合）、分区表（命中矩阵） |
| **因果推断**     | EconML (`CausalForestDML`, `PolicyTree`) / CausalML | Tier 2 算法层，API 或离线调度                 |
| **监督学习**     | scikit-learn (`DecisionTreeClassifier`) / LightGBM  | Tier 1 算法层，本地 Python                 |
| **特征工程**     | Pandas / NumPy / category\_encoders                 | 分箱、Target Encoding、WoE               |
| **大模型**      | OpenAI 兼容协议 API                                     | 意图理解、报告生成、DSL 生成、SOP 编排              |
| **NLP/NER**  | 轻量级隐私识别模型（本地部署）                                     | DLP 第二层                              |
| **版本控制**     | Git（GitPython / subprocess）                         | Wiki 知识网物理存储                         |
| **图数据库（可选）** | Neo4j 或 PostgreSQL + 关系表                            | 知识图谱索引辅轨                             |
| **任务调度**     | Celery / APScheduler                                | 因果森林训练、黄金评测集更新、Wiki 聚合               |
| **消息队列**     | Redis / RabbitMQ                                    | 跨智能体信号、标签回传                          |
| **前端**       | React / Vue 3                                       | 管理界面、确认卡片、对抗可视化                      |

### 1.4 异步任务架构



系统中多个操作属于**长时任务**（秒级到小时级），若以同步 HTTP 请求处理会导致网关超时和前端阻塞。所有长时操作必须通过异步任务队列调度：



| 操作                 | 预估耗时          | 调度方式               | Worker           |
| ------------------ | ------------- | ------------------ | ---------------- |
| Tier 2 因果森林训练      | 10 min \~ 2 h | Celery async task  | `causal-trainer` |
| Tier 1 监督式决策树训练    | 30 s \~ 5 min | Celery async task  | `sandbox-worker` |
| 沙盒跑批（单轮）           | 1 \~ 5 min    | Celery async task  | `sandbox-worker` |
| 红蓝对抗循环（多轮沙盒 + LLM） | 3 \~ 15 min   | Celery chain/chord | `sandbox-worker` |
| 黄金评测集刷新            | 10 \~ 30 min  | 定时任务 (beat)        | `causal-trainer` |
| 标签回传消费 + Uplift 计算 | 1 \~ 10 min   | 定时 / 事件触发          | `sandbox-worker` |
| Wiki 衰减检测 + 聚合摘要   | 1 \~ 5 min    | 定时任务 (beat)        | `wiki-service`   |



**技术实现**：



```python
# shared/tasks/celery_app.py
from celery import Celery

app = Celery("risk_control", broker="redis://redis:6379/0", backend="redis://redis:6379/1")
app.conf.update(
    task_serializer="json",
    result_expires=3600 * 24,
    task_acks_late=True,               # 任务完成后才确认，worker 崩溃可重试
    worker_prefetch_multiplier=1,      # 长任务逐条取，避免饥饿
    task_time_limit=7200,              # 全局硬超时 2h
    task_soft_time_limit=6000,         # 软超时 100min，允许优雅清理
)
```



**API 层异步交互协议**：



前端发起长时操作时，API 立即返回 `task_id`，前端通过轮询或 WebSocket 获取进度：



```plain&#x20;text
POST /api/adversarial/start
Response: { "task_id": "abc-123", "status": "QUEUED" }

GET /api/tasks/{task_id}/status
Response: { "task_id": "abc-123", "status": "RUNNING", "progress": {"round": 2, "total_rounds": 3}, "eta_seconds": 120 }

GET /api/tasks/{task_id}/result
Response: { "task_id": "abc-123", "status": "COMPLETED", "result": { ... scorecard ... } }
```



**任务状态表**：



```sql
CREATE TABLE async_tasks (
    task_id         VARCHAR(64)  PRIMARY KEY,
    task_type       VARCHAR(64)  NOT NULL,
    submitted_by    VARCHAR(64)  NOT NULL,
    status          VARCHAR(16)  NOT NULL DEFAULT 'QUEUED'
        CHECK (status IN ('QUEUED','RUNNING','COMPLETED','FAILED','CANCELLED')),
    progress        JSONB,
    result          JSONB,
    error_message   TEXT,
    submitted_at    TIMESTAMPTZ  NOT NULL DEFAULT NOW(),
    started_at      TIMESTAMPTZ,
    completed_at    TIMESTAMPTZ
);
```



**前端交互模式**：用户点击"开始对抗"后，前端展示"对抗进行中"进度面板（含当前轮次、已用时间、预估剩余时间），用户可继续使用系统其他功能，对抗完成后通过消息通知回调。

### 2.1 关系型表（PostgreSQL DDL）



#### `risk_config` — 统一配置管理



```sql
CREATE TABLE risk_config (
    id              BIGSERIAL PRIMARY KEY,
    config_key      VARCHAR(128) NOT NULL,
    business_line   VARCHAR(64)  NOT NULL DEFAULT '__GLOBAL__',
    scene           VARCHAR(64)  NOT NULL DEFAULT '__ANY__',
    campaign_period VARCHAR(64)  NOT NULL DEFAULT '__ANY__',
    value           TEXT         NOT NULL,
    value_type      VARCHAR(16)  NOT NULL CHECK (value_type IN ('float','int','string','json')),
    updated_by      VARCHAR(64)  NOT NULL,
    updated_at      TIMESTAMPTZ  NOT NULL DEFAULT NOW(),
    version         INT          NOT NULL DEFAULT 1,
    is_deleted      BOOLEAN      NOT NULL DEFAULT FALSE,
    UNIQUE (config_key, business_line, scene, campaign_period)
);

CREATE INDEX idx_rc_lookup ON risk_config (config_key, business_line, scene, campaign_period)
    WHERE is_deleted = FALSE;

COMMENT ON TABLE risk_config IS '自建配置管理模块，替代外部配置中心。优先级：(line+scene+campaign) > (line+scene) > (line) > 全局';
```



#### `rules` — 规则元数据



```sql
CREATE TABLE rules (
    id              BIGSERIAL PRIMARY KEY,
    rule_id         VARCHAR(64)  NOT NULL UNIQUE,
    rule_version    INT          NOT NULL DEFAULT 1,
    source          VARCHAR(32)  NOT NULL CHECK (source IN ('CANDIDATE','ONLINE','RETIRED')),
    tier            VARCHAR(16)  NOT NULL CHECK (tier IN ('tier2','tier1','tier0')),
    business_line   VARCHAR(64)  NOT NULL,
    scene           VARCHAR(64),
    rule_json       JSONB        NOT NULL,
    dsl_text        TEXT,
    candidate_json  JSONB,
    created_by      VARCHAR(64)  NOT NULL,
    created_at      TIMESTAMPTZ  NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMPTZ  NOT NULL DEFAULT NOW(),
    status          VARCHAR(16)  NOT NULL DEFAULT 'ACTIVE'
        CHECK (status IN ('ACTIVE','DISABLED','RETIRED'))
);

CREATE INDEX idx_rules_active ON rules (source, status) WHERE status = 'ACTIVE';

COMMENT ON COLUMN rules.rule_json IS '结构化规则 JSON，格式见 §2.3 Rule JSON Schema';
COMMENT ON COLUMN rules.candidate_json IS '产生该规则的原始候选 JSON（Tier 2/1/0），用于 Semantic Validator 比对锚点';
```

**Wiki 实体页写锁实现**：



```python
# shared/wiki/write_lock.py
import redis
import time

class WikiWriteLock:
    """
    基于 Redis 的实体页级分布式互斥锁。
    同一场景实体页同一时刻仅允许一个写入操作（产品文档 §5.2 约束）。
    多个验收通过的方案按 verified_at 排队，FIFO 执行。
    """
    def __init__(self, redis_client: redis.Redis, lock_timeout: int = 300):
        self.redis = redis_client
        self.lock_timeout = lock_timeout

    def acquire(self, scene_key: str, request_id: str, wait_timeout: int = 600) -> bool:
        lock_key = f"wiki_write_lock:{scene_key}"
        deadline = time.time() + wait_timeout
        while time.time() < deadline:
            if self.redis.set(lock_key, request_id, nx=True, ex=self.lock_timeout):
                return True
            time.sleep(1)
        return False

    def release(self, scene_key: str, request_id: str):
        lock_key = f"wiki_write_lock:{scene_key}"
        pipe = self.redis.pipeline(True)
        pipe.watch(lock_key)
        if pipe.get(lock_key) == request_id.encode():
            pipe.multi()
            pipe.delete(lock_key)
            pipe.execute()
```

```python
# pipeline/gate/maturity_checker.py 核心逻辑
class GateResult:
    tier: Literal["tier2", "tier1", "tier0"]
    reason: str
    metrics: dict  # 各检查项的实际值

def check_maturity(dataset_meta: dict, config: RiskConfig) -> GateResult:
    """
    检查项及阈值均从 risk_config 读取（config_key 前缀 gate_）
    """
    has_labels = dataset_meta["label_coverage"] >= config.get("gate_label_coverage_min")  # 如 0.6
    enough_samples = (
        dataset_meta["total_labeled"] >= config.get("gate_min_samples")          # 如 500
        and dataset_meta["black_count"] >= config.get("gate_min_black_samples")  # 如 100
    )
    has_treatment = dataset_meta["has_treatment_field"]
    has_overlap = dataset_meta["propensity_overlap"] >= config.get("gate_overlap_min")  # 如 0.3

    if has_labels and enough_samples and has_treatment and has_overlap:
        return GateResult(tier="tier2", ...)
    elif has_labels and enough_samples:
        return GateResult(tier="tier1", reason="缺干预记录或对照组重叠不足")
    else:
        return GateResult(tier="tier0", reason="标签不可用或样本量不足")
```

```python
from sklearn.tree import DecisionTreeClassifier

def train_and_export(X, y, config: RiskConfig) -> RuleCandidateJSON:
    clf = DecisionTreeClassifier(
        max_depth=config.get("tier1_max_depth", 4),
        min_samples_leaf=config.get("tier1_min_samples_leaf", 50),
        class_weight="balanced"
    )
    clf.fit(X, y)
    return export_tree_to_json(clf, source="supervised_tree")

def export_tree_to_json(clf, source: str) -> dict:
    tree = clf.tree_
    nodes = []
    for i in range(tree.node_count):
        if tree.children_left[i] == tree.children_right[i]:  # leaf
            prob = tree.value[i][0][1] / tree.value[i][0].sum()
            nodes.append({
                "id": i, "type": "leaf",
                "class_probability": round(prob, 4),
                "causal_effect": None,
                "sample_size": int(tree.n_node_samples[i])
            })
        else:  # split
            nodes.append({
                "id": i, "type": "split",
                "feature": feature_names[tree.feature[i]],
                "threshold": round(tree.threshold[i], 4),
                "causal_effect": None,
                "class_probability": None,
                "left": tree.children_left[i],
                "right": tree.children_right[i]
            })
    return {"tree_id": generate_id("ST"), "source": source, "nodes": nodes}
```

```python
# pipeline/sandbox/rule_parser.py
def rule_json_to_sql(rule_json: dict) -> tuple[str, list]:
    """
    返回 (WHERE 子句, 参数列表)，使用参数绑定防注入
    """
    logic = rule_json["logic"]  # AND / OR
    clauses = []
    params = []

    for cond in rule_json.get("conditions", []):
        field = _validate_field(cond["field"])  # 白名单校验字段名
        op = cond["op"]
        val = cond["value"]

        if op == "IN":
            placeholders = ", ".join(["%s"] * len(val))
            clauses.append(f"{field} IN ({placeholders})")
            params.extend(val)
        elif op == "BETWEEN":
            clauses.append(f"{field} BETWEEN %s AND %s")
            params.extend(val)
        else:
            clauses.append(f"{field} {op} %s")
            params.append(val)

    for group in rule_json.get("groups", []):
        sub_sql, sub_params = rule_json_to_sql(group)
        clauses.append(f"({sub_sql})")
        params.extend(sub_params)

    connector = f" {logic} "
    return connector.join(clauses), params
```

### 6.4 业财成绩单聚合公式



```sql

WITH hits AS (
    SELECT rule_id, sample_label, is_hit, sample_id
    FROM hit_matrix WHERE run_id = %(run_id)s
),
black_hits AS (
    SELECT COUNT(*) AS hit_black_count,
           COALESCE(SUM(gs.amount), 0) AS black_hit_amount
    FROM hits h JOIN golden_samples gs ON h.sample_id = gs.sample_id
    WHERE h.sample_label = 'BLACK' AND h.is_hit = TRUE AND h.rule_id = %(rule_id)s
),
white_hits AS (
    SELECT COUNT(*) AS hit_white_count,
           COALESCE(SUM(gs.amount), 0) AS white_hit_amount
    FROM hits h JOIN golden_samples gs ON h.sample_id = gs.sample_id
    WHERE h.sample_label = 'WHITE' AND h.is_hit = TRUE AND h.rule_id = %(rule_id)s
),
totals AS (
    SELECT COUNT(*) FILTER (WHERE label = 'WHITE') AS total_white,
           COUNT(*) FILTER (WHERE label = 'BLACK') AS total_black
    FROM golden_samples WHERE dataset_version = %(dataset_version)s
)
SELECT
    bh.hit_black_count,
    wh.hit_white_count,
    ROUND(wh.hit_white_count::numeric / NULLIF(t.total_white, 0), 6) AS white_user_kill_rate,
    ROUND(bh.black_hit_amount * %(recover_ratio)s, 2) AS gmv_saved,
    ROUND(wh.white_hit_amount * %(loss_ratio)s, 2) AS gmv_lost_estimate
FROM black_hits bh, white_hits wh, totals t;
```



其中 `recover_ratio` 和 `loss_ratio` 从 `risk_config` 表按业务线读取。

```python
class AdversarialDecision(BaseModel):
    accepted: bool
    violations: list[str]
    scorecard: BusinessScorecard

def validate(scorecard: BusinessScorecard, tolerance: ToleranceConfig) -> AdversarialDecision:
    violations = []

    if scorecard.white_user_kill_rate > tolerance.fpr_limit:
        violations.append(
            f"FPR {scorecard.white_user_kill_rate:.4f} 超限 {tolerance.fpr_limit:.4f}"
        )

    if scorecard.gmv_saved > 0:
        loss_ratio = scorecard.gmv_lost_estimate / scorecard.gmv_saved
        if loss_ratio > tolerance.gmv_loss_ratio_limit:
            violations.append(
                f"折损/挽损比 {loss_ratio:.4f} 超限 {tolerance.gmv_loss_ratio_limit:.4f}"
            )

    return AdversarialDecision(
        accepted=len(violations) == 0,
        violations=violations,
        scorecard=scorecard
    )
```

```python
from enum import Enum

class LabelReadiness(Enum):
    OBSERVING = "observing"           # 标签未回传，继续观察
    INSUFFICIENT = "insufficient"     # 标签部分回传，样本量不足
    READY = "ready"                   # 标签充分回传，可评估
    TIMEOUT = "timeout"               # 超过最大等待周期仍不足

def check_label_readiness(rule_id: str) -> LabelReadiness:
    hit_count = monitor_log_repo.count_hits(rule_id)
    labeled_count = label_repo.count_labeled(rule_id)
    max_wait_days = int(config_reader.get_cascaded("stage3_max_wait_days", ...))
    min_confidence_samples = int(config_reader.get_cascaded("stage3_min_confidence_samples", ...))
    days_since_stage2 = lifecycle_repo.days_in_stage(rule_id, "STAGE3")

    if labeled_count == 0:
        if days_since_stage2 > max_wait_days:
            return LabelReadiness.TIMEOUT
        return LabelReadiness.OBSERVING

    coverage = labeled_count / max(hit_count, 1)
    if coverage < 0.6 or labeled_count < min_confidence_samples:
        if days_since_stage2 > max_wait_days:
            return LabelReadiness.TIMEOUT
        return LabelReadiness.INSUFFICIENT

    return LabelReadiness.READY
```



**标签消费与影子收益计算**（仅在 READY 状态触发）：



```python
def consume_labels(rule_id: str):
    """幂等消费标签，以 (user_id, event_id, label_timestamp) 去重"""
    labels = label_client.pull(rule_id, idempotent_key=("user_id","event_id","label_timestamp"))
    label_repo.upsert(labels)

def settle_shadow_revenue(rule_id: str) -> SettlementResult | str:
    consume_labels(rule_id)
    readiness = check_label_readiness(rule_id)

    if readiness == LabelReadiness.OBSERVING:
        return "标签未回传，继续观察"
    if readiness == LabelReadiness.INSUFFICIENT:
        return "样本量不足，建议延长观察"
    if readiness == LabelReadiness.TIMEOUT:
        alert_service.escalate(rule_id, "标签等待超时，升级至人工决策")
        return "超时，已升级人工决策"

    # READY: 标签充分，执行结算
    hit_logs = monitor_log_repo.get_hits(rule_id)
    labels_map = label_repo.get_labels_map(rule_id)  # {user_id: label}

    shadow_saved = sum(
        log.amount for log in hit_logs
        if labels_map.get(log.user_id) == "BLACK"
    )
    shadow_killed = sum(
        1 for log in hit_logs
        if labels_map.get(log.user_id) == "WHITE"
    )
    unlabeled = sum(
        1 for log in hit_logs
        if log.user_id not in labels_map
    )

    uplift, ci_lower, ci_upper = calculate_uplift_with_ci(rule_id, confidence=0.95)

    return SettlementResult(
        shadow_saved=shadow_saved,
        shadow_kill_count=shadow_killed,
        unlabeled_count=unlabeled,
        label_coverage=1 - unlabeled / max(len(hit_logs), 1),
        uplift=uplift,
        ci_95=(ci_lower, ci_upper),
        significant=ci_lower > 0
    )
```

产品要求手工上传通道限制 `< 5MB 或 2000 行`。若仅在应用层拦截，恶意或误传的超大文件（数百 MB）已消耗带宽和内存，可能打满服务器导致其他用户无法登录。因此必须实施**多层拦截**：



```plain&#x20;text
┌────────────────────────────────────────────────────────┐
│  第一层：反向代理（Nginx / ALB / 云 LB）                 │
│  配置 client_max_body_size 5m;                         │
│  超限 → 413 立即断连，请求体不进入后端                    │
└────────────────────────┬───────────────────────────────┘
                         │ ≤ 5MB
┌────────────────────────▼───────────────────────────────┐
│  第二层：FastAPI 中间件（流式读取）                       │
│  边读边计行数，超 2000 行立即 abort 返回 400             │
│  防止 5MB 内但行数爆炸（如单行 1 字节 × 500万行）        │
└────────────────────────┬───────────────────────────────┘
                         │ ≤ 5MB & ≤ 2000 行
┌────────────────────────▼───────────────────────────────┐
│  第三层：文件类型白名单校验                               │
│  仅允许 .csv / .tsv / .xlsx，拒绝可执行文件              │
└────────────────────────┬───────────────────────────────┘
                         │ 校验通过
                         ▼
                    进入 DLP 网关脱敏
```

```python
# shared/config/reader.py
def get_cascaded(config_key: str, business_line: str, scene: str = None, campaign: str = None) -> str:
    """
    查询优先级：(key+line+scene+campaign) > (key+line+scene) > (key+line) > (key+__GLOBAL__)
    """
    candidates = [
        (config_key, business_line, scene or "__ANY__", campaign or "__ANY__"),
        (config_key, business_line, scene or "__ANY__", "__ANY__"),
        (config_key, business_line, "__ANY__", "__ANY__"),
        (config_key, "__GLOBAL__", "__ANY__", "__ANY__"),
    ]
    for key, bl, sc, cp in candidates:
        result = db.query(
            "SELECT value FROM risk_config WHERE config_key=%s AND business_line=%s "
            "AND scene=%s AND campaign_period=%s AND is_deleted=FALSE",
            [key, bl, sc, cp]
        )
        if result:
            return result.value
    raise ConfigNotFoundError(f"{config_key} not found for {business_line}/{scene}/{campaign}")
```

| `E3001` | 沙盒 OOM / 超时           | 终止 + 提示抽样                 |

### 11.3 降级决策树



```plain&#x20;text
flowchart TD
    Start[异常发生] --> IsSecurity{涉及安全/隐私?}
    IsSecurity -->|是: DLP/脱敏| Block[阻断流程, 不降级]
    IsSecurity -->|否| IsRetryable{可重试?}
    IsRetryable -->|是| Retry[按策略重试]
    Retry --> RetryOK{重试成功?}
    RetryOK -->|是| Continue[继续流程]
    RetryOK -->|否| IsCore{核心路径?}
    IsCore -->|是: 沙盒/对抗| FailFast[快速失败 + 明确告知]
    IsCore -->|否: 偏好/模板| Degrade[降级: 跳过功能 + 提示]
```
